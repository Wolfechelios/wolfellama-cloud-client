# WolfeLlama Product Positioning

## Product Name

WolfeLlama Cloud Client

## One-Line Positioning

A private local/cloud AI operator for builders who want one desktop workspace to chat with models, remember project context, plan larger tasks, and edit GitHub code safely.

## Short Pitch

WolfeLlama is not just another chatbot. It is a personal AI workbench that connects local Ollama models, cloud models, project memory, Agent Mode planning, and GitHub code editing into one focused desktop-style interface.

The selling point is control: the user can work locally, switch to cloud models when needed, keep memory across sessions, plan large tasks, and push code changes without jumping between five different tools.

## Target Buyer

Primary buyer:

- Solo builders.
- AI hobbyists.
- Local LLM users.
- Indie hackers.
- Developers who use GitHub but want a simpler AI control panel.
- Technical workers building projects with ChatGPT, Cursor, Bolt, Lovable, Ollama, and GitHub.

Secondary buyer:

- Small business operators who want private AI workflows.
- Field technicians learning to automate notes, code, documents, and workflows.
- Power users who want local + cloud AI without being locked into one provider.

## Main Pain

The buyer is tired of scattering work across:

- ChatGPT.
- Ollama terminal.
- GitHub browser editor.
- Local code editor.
- AI coding tools.
- Project notes.
- Random memory files.

WolfeLlama brings that into one place.

## High-Value Selling Point

The strongest selling point:

> Build with AI without losing control of your models, memory, code, or workflow.

## Differentiator

Most AI coding apps are either cloud-first or editor-first.

WolfeLlama is control-first:

- Local model support through Ollama.
- Cloud model support through OpenAI-compatible providers.
- Terminal Mode for raw local model use.
- Memory for continuing work later.
- Agent Mode for larger delegated tasks.
- GitHub code editing for direct repo changes.
- Hardware/workflow expansion path.

## Product Category

Private AI development operator.

Alternative phrases:

- Local/cloud AI workbench.
- AI project command room.
- Personal AI coding operator.
- Desktop AI control center.

## Early Access Promise

Use one app to talk to local/cloud models, remember your work, plan larger tasks, and edit GitHub code directly.

## What Not To Sell It As Yet

Do not call it a fully autonomous coding agent yet.

Current honest language:

- Agent Mode foundation.
- Step-by-step task planner.
- GitHub-backed code editor.
- Local/cloud model workspace.

Future language after tools mature:

- Autonomous coding operator.
- Project-aware AI builder.
- AI developer assistant with safe execution.
