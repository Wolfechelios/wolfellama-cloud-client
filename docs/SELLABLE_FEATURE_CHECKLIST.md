# Sellable Feature Checklist

This file defines the feature work that increases WolfeLlama's per-copy selling value.

## Tier 1: $29 Early Access

Required:

- Stable startup.
- Simple Command Room UI.
- Ollama local chat.
- Terminal Mode.
- Cloud provider path.
- Persistent chat memory.
- Agent Mode planning.
- GitHub code editor component.
- Basic README and install instructions.

Current status: mostly present, code editor mount still needs completion.

## Tier 2: $79 Stable V1

Required:

- Mac app package.
- Windows app package.
- First-run onboarding.
- Clean provider setup screen.
- Code Editor mounted in sidebar.
- GitHub file load/edit/save working in app.
- Clear warning before saving to GitHub.
- Task export/import.
- Better error messages.
- Stable visual polish.

## Tier 3: $149 Pro

Required:

- Project file inventory.
- Read-only repo/file search.
- Patch preview before commit.
- Approval queue.
- Multi-file edit plan.
- Verification checklist.
- Build/test command preview.
- Project memory notes.
- Multi-repo switching.

## Tier 4: Subscription / Business

Required:

- Secure key storage.
- Team profiles.
- Shared workspaces.
- Provider usage tracking.
- Advanced audit reports.
- Installer signing.
- Update channel.
- Documentation site.
- Support and licensing flow.

## Highest-Value Feature Order

1. Mount GitHub Code Editor in the UI.
2. Add warning and preview before commit.
3. Add first-run onboarding.
4. Add packaged Mac app.
5. Add packaged Windows app.
6. Add project memory notes.
7. Add patch preview.
8. Add verification report.
9. Add command preview.
10. Add secure key storage.

## Rule

Do not add visual complexity unless it helps the buyer complete one of these actions:

- Chat with local/cloud AI.
- Remember work.
- Plan a larger task.
- Edit code.
- Commit code.
- Verify work.
